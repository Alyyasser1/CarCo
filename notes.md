PROJECT: CarCo
STACK: React + TypeScript + Firebase + React Router

WHAT IT DOES:
A branded AI-powered web app (not designed to look like a chatbot) with two main services:

1. Car Recommendation — user inputs preferences (brand, category, year, requirements)
   and the AI recommends cars matching their needs
2. Car Diagnosis — user inputs car brand, year, and problem description and the AI
   diagnoses the issue and provides solutions

AUTH:

- Firebase Authentication
- Email/Password + Google Sign-in + Phone (OTP)
- Auth UI is a POPUP/MODAL not a separate page
- Users can use the services WITHOUT being logged in
- Logged in users get their chat history saved to Firebase
- Logged out users can still use the app but chats are not saved

PAGES (React Router):

- AuthPage "/" → landing/hero page, has Login + Signup buttons that open modal
- RecommendationPage "/recommend" → car recommendation service
- DiagnosisPage "/diagnose" → car diagnosis service

DATA ENTITIES:

- User → handled entirely by Firebase Auth, not defined manually
- SavedSession → defined in firestore.ts, contains:
  - id (auto-generated by Firestore)
  - userId (links session to Firebase Auth user)
  - type ("recommendation" | "diagnosis")
  - title (requirements/problem text, max 80 chars)
  - createdAt (Firebase serverTimestamp)
  - result (RecommendationResult | DiagnosisResult)

FIRESTORE STRUCTURE:
sessions/ ← flat collection
├── auto-id/ ← one session per document
│ userId, type, title, createdAt, result
└── ...
All users share one collection. userId field filters per user.
